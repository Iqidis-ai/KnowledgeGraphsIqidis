Database Schema

The core data models defined in `src/core/storage/models.py`. Understanding these structures is key to understanding the graph capabilities.

Entity Model

table: entities
id UUID (PK)

Unique identifier for the node.

type VARCHAR

E.g., "Person", "Organization", "Event".

canonical_name VARCHAR

The main display name.

properties JSON

Flexible key-value pairs (e.g. {"age": 50}).

Edge Model

table: edges
source_entity_id UUID (FK)

Origin node of the relationship.

target_entity_id UUID (FK)

Destination node of the relationship.

relation_type VARCHAR

E.g., "WORKS_FOR", "SIGNED".

provenance_span JSON

Where in the doc this was found.
